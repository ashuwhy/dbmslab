2. Schema Design

The system uses a PostgreSQL database with the following tables. The schema supports universities, programs, courses, students, instructors, enrollments, teaching assignments, content items, and role-based access control.

2.1 RBAC and Users

TableDescription
app_userRoles-based access control: id (PK), email (unique), password_hash, role (admin/instructor/student/analyst), created_at. Links to student.email or instructor.email.

2.2 Core Domain Tables

TableKey Columns
universityuniversity_id (PK), name (unique), country
programprogram_id (PK), program_name, program_type, duration_weeks_or_months
coursecourse_id (PK), course_name (unique), duration_weeks, university_id (FK), program_id (FK), textbook_id (FK)
textbooktextbook_id (PK), title, isbn (unique), url
topictopic_id (PK), topic_name (unique)
course_topiccourse_id (FK, PK), topic_id (FK, PK)

2.3 People and Assignments

TableKey Columns
studentstudent_id (PK), email (unique), full_name, age (≥13), country, category, skill_level
instructorinstructor_id (PK), full_name, email (unique)
enrollmentstudent_id (FK, PK), course_id (FK, PK), enroll_date, evaluation_score
teaching_assignmentinstructor_id (FK, PK), course_id (FK, PK), role
content_itemcontent_id (PK), course_id (FK), content_type, title, url

2.4 Relationships (Summary)

  • Course belongs to one University and one Program; has one Textbook; many ContentItems; many Enrollments; many TeachingAssignments; many Topics via course_topic.
  • Student has many Enrollments; email links to app_user for login.
  • Instructor has many TeachingAssignments; email links to app_user for login.
  • Enrollment is a many-to-many between Student and Course with enroll_date and evaluation_score.
  • Teaching assignment is a many-to-many between Instructor and Course with role.

3. Languages and Technology Stack

3.1 Overview

  • Frontend: Next.js (App Router), React, TypeScript
  • Backend: FastAPI (Python 3.11+)
  • Database: PostgreSQL
  • Build/Tooling: Make, uv (or pip), npm

3.2 Backend (API)

  • Language: Python 3.11+
  • Framework: FastAPI
  • ORM / DB: SQLAlchemy 2.x (async), asyncpg (PostgreSQL driver)
  • Validation & settings: Pydantic, pydantic-settings
  • Auth: JWT (python-jose), password hashing (passlib/bcrypt)
  • Server: Uvicorn

3.3 Frontend (Web)

  • Language: TypeScript
  • Framework: Next.js 16 (App Router)
  • UI: React 19, Tailwind CSS 4, Radix UI (dialogs, labels, scroll-area, etc.), Lucide React (icons)
  • Linting: ESLint, eslint-config-next

3.4 Project Structure

  • /apps/api – FastAPI backend (routers: auth, student, instructor, admin, analyst)
  • /apps/web – Next.js frontend (admin, analyst, instructor, student, login)
  • /docs – Documentation (API.md, SCHEMA.md) and this submission document
